## Decisions Made

- Standalone `compute-season.ts` script vs inlining in `generate-demo.ts` — avoids circular dependency between generate-demo (needs politicians.json) and score-all (needs season-weeks.json)
- `config.ts` reads JSON synchronously at module load (`readFileSync`) — consistent with how SEASON_WEEKS is consumed (synchronous const import), safe in all Node.js script contexts
- Hardcoded fallback dates kept in catch block — ensures `loadSeasonWeeks()` is safe to call before compute-season has run on a fresh checkout
