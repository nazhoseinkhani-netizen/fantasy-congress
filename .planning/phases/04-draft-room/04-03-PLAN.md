Create `src/components/draft/post-draft.tsx` — 'use client'.

Props: `politicians: Map<string, Politician>`

This component reads from `useDraftStore` (teams, picks) and `useGameStore` (to save roster).

**On mount (useEffect, runs once when phase === 'complete'):**
1. Compute grades for all 4 teams by calling `gradeDraftTeam()` for each team's roster (look up Politician objects from the map).
2. Call `generateGradeWriteup()` for each grade.
3. Call `findSleeperPicks()` on all picks.
4. Store computed grades and sleepers in local useState.
5. **Save user's drafted roster to game store**: Get the user's team from `teams[userTeamIndex]`. The first `DRAFT_CONFIG.ACTIVE_SLOTS` (3) picks become active, the remaining `DRAFT_CONFIG.BENCH_SLOTS` (1) become bench. Get the user's team ID from the demo state (or generate a stable one like `draft-team-${draftSessionId}`). Call game store update: `useGameStore.getState().swapRosterSlots` won't work here since we're replacing the whole roster — instead, directly set `rosterOverrides` by reading current state and spreading: `useGameStore.setState(state => ({ rosterOverrides: { ...state.rosterOverrides, [userTeamId]: { active: activeIds, bench: benchIds } } }))`. Actually, since game-store doesn't expose a generic setter, the simplest approach: read the draft store's user team roster, split into active (first 3) and bench (last 1), and use `useGameStore.setState` directly to update `rosterOverrides` for the user's team ID from the demo state (loaded via `loadDemoState()` — pass userTeamId as a prop or load it in this component).
