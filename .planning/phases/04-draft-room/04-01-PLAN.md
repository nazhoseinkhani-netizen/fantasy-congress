Create `src/lib/draft/ai-engine.ts`:

4 AI archetypes with scoring weights applied to politician attributes:
- `value-hunter`: Maximizes seasonPoints / salaryCap ratio. Weights: seasonPoints * 0.4 + (seasonPoints / salaryCap * 10000) * 0.6
- `corruption-chaser`: Prefers high insiderRiskScore. Weights: insiderRiskScore * 0.6 + seasonPoints * 0.4
- `party-loyalist`: Prefers one party (set at creation — first 2 picks determine party preference, then +30% bonus to same party). Weights: seasonPoints * 0.7 + (sameParty ? seasonPoints * 0.3 : 0)
- `balanced`: Equal weight to points and value. Weights: seasonPoints * 0.5 + (seasonPoints / salaryCap * 10000) * 0.3 + insiderRiskScore * 0.2

Export `scoreForArchetype(archetype: AIArchetype, politician: Politician, teamRoster: string[], allPoliticians: Map<string, Politician>): number` and `selectAIPick(archetype: AIArchetype, availableIds: string[], teamRoster: string[], salaryRemaining: number, politicians: Map<string, Politician>): string`.

`selectAIPick` filters to affordable politicians, scores each, sorts descending, then applies mistake logic: `Math.random() < 0.10` pick #2, `< 0.15` pick #3, else pick #1. If filtered list is empty (can't afford anyone), return the cheapest available politician regardless.

**State interface (DraftState):**
```typescript
interface DraftState {
  phase: DraftPhase           // 'lobby' | 'countdown' | 'drafting' | 'user-turn' | 'complete'
  teams: DraftTeam[]          // 4 teams (1 human + 3 AI)
  picks: DraftPick[]          // All picks made so far
  availablePool: string[]     // bioguideIds not yet drafted
  currentPickIndex: number    // 0 to TOTAL_PICKS-1
  userTeamIndex: number       // Which of the 4 teams is the user (randomized 0-3)
  userPickTimer: number       // Seconds remaining for user pick (60 -> 0)
  isAITurnPending: boolean    // Prevents duplicate setTimeout scheduling
  draftSessionId: string      // Random string to detect stale sessions
}
```

**Actions interface (DraftActions):**
```typescript
interface DraftActions {
  initDraft: (politicians: Politician[], leagueTeams: Team[]) => void
  startCountdown: () => void
  startDrafting: () => void
  recordPick: (bioguideId: string, salaryCap: number) => void
  setUserPickTimer: (seconds: number) => void
  setAITurnPending: (pending: boolean) => void
  resetDraft: () => void
  getCurrentTeamIndex: () => number
  isUserTurn: () => boolean
}
```

**initDraft implementation:**
1. Generate random userTeamIndex (0-3)
2. Create 4 DraftTeam objects. Use team names from the passed leagueTeams parameter (e.g., "Filibusters", "Capitol Gains", etc.). The user's team keeps its name; AI teams get archetypes assigned round-robin from `['value-hunter', 'corruption-chaser', 'party-loyalist', 'balanced']` (excluding the user's index).
3. Set availablePool to all politician bioguideIds from the passed politicians array
4. Set phase to 'lobby', currentPickIndex to 0, picks to [], userPickTimer to DRAFT_CONFIG.USER_PICK_TIMER_SECONDS
5. Generate a new draftSessionId via `Date.now().toString(36)`

**recordPick implementation:**
1. Create DraftPick with current pickNumber, round (Math.floor(currentPickIndex / TEAM_COUNT)), teamIndex from snakeDraftTeamIndex, bioguideId, salaryCap, timestamp
2. Update the team's roster array (push bioguideId) and salaryUsed (+= salaryCap)
3. Remove bioguideId from availablePool
4. Increment currentPickIndex
5. If currentPickIndex >= TOTAL_PICKS, set phase to 'complete'
6. Else: determine next team via snakeDraftTeamIndex. If next team is user, set phase to 'user-turn' and reset userPickTimer to 60. If next team is AI, set phase to 'drafting' and isAITurnPending to false.

**getCurrentTeamIndex:** Returns `snakeDraftTeamIndex(get().currentPickIndex, DRAFT_CONFIG.TEAM_COUNT)`

**isUserTurn:** Returns `getCurrentTeamIndex() === get().userTeamIndex`

**resetDraft:** Clears all state back to initial values with phase = 'lobby'.

**Persist partialize:** Only persist `phase`, `teams`, `picks`, `availablePool`, `currentPickIndex`, `userTeamIndex`, `draftSessionId`. Do NOT persist `isAITurnPending` or `userPickTimer` (runtime-only).
