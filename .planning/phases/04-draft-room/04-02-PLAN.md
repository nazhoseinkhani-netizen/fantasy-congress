**AI Turn Orchestration** (critical — per D-01, D-04, RESEARCH pitfall 5):
In draft-board.tsx, use a useEffect that watches `draftState.phase` and `draftState.currentPickIndex`. When phase is `'drafting'` (meaning it's an AI turn) AND `!draftState.isAITurnPending`:
1. Call `setAITurnPending(true)` immediately
2. Get current team via `getCurrentTeamIndex()`
3. Calculate random delay: `DRAFT_CONFIG.AI_DELAY_MIN_MS + Math.random() * (DRAFT_CONFIG.AI_DELAY_MAX_MS - DRAFT_CONFIG.AI_DELAY_MIN_MS)`
4. Set a timeout. Store the timeout ID in a `useRef<NodeJS.Timeout>`.
5. When timeout fires: check `useDraftStore.getState().draftSessionId` matches the session ID captured at effect start (stale check). If stale, do nothing. Otherwise call `selectAIPick(team.archetype, availablePool, team.roster, DRAFT_CONFIG.SALARY_CAP - team.salaryUsed, politicianMap)` to get the bioguideId, look up the politician's salaryCap, then call `recordPick(bioguideId, salaryCap)`.
6. Cleanup: on useEffect return, clear the timeout via `clearTimeout(timeoutRef.current)`.

**User Pick Timer** (per D-03):
Separate useEffect: When phase is `'user-turn'`, start a 1-second setInterval that decrements `userPickTimer`. When timer hits 0, auto-pick the best available politician (highest seasonPoints that fits salary cap) via `recordPick`. Clean up interval on unmount or phase change.
