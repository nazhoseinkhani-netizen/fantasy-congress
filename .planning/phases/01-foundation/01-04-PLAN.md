    1. Create scripts/validate-photos.ts:
       - Read _raw-politicians.json
       - For each politician, try photo URLs in order:
         a. `https://bioguide.congress.gov/bioguide/photo/${bioguideId[0]}/${bioguideId}.jpg` (Congress.gov bioguide)
         b. `https://unitedstates.github.io/images/congress/450x550/${bioguideId}.jpg` (unitedstates/images CDN)
         c. `https://unitedstates.github.io/images/congress/original/${bioguideId}.jpg` (original size fallback)
       - For each URL, do HTTP HEAD request with 5-second timeout. If 200, use that URL.
       - If all fail, generate an initials SVG data URI:
         ```
         data:image/svg+xml,<svg xmlns="http://www.w3.org/2000/svg" width="200" height="200"><rect fill="${partyColor}" width="200" height="200"/><text x="50%" y="50%" fill="white" font-size="72" font-family="sans-serif" text-anchor="middle" dominant-baseline="central">${initials}</text></svg>
         ```
         Where partyColor = D: '#3B82F6', R: '#EF4444', I: '#22C55E'
       - Rate limit: max 10 concurrent requests (use a simple semaphore or batch in groups of 10)
       - Log results: { validated: number, fallbackUsed: number, initialsGenerated: number }
       - Write updated politicians with validated photoUrl to `public/data/_validated-politicians.json`

    2. Create scripts/score-all.ts:
       - Read `_validated-politicians.json` and `_raw-trades.json`
       - Import `scoreTrade`, `scorePolitician` from `@/lib/scoring/engine` (configure tsx path aliases or use relative import `../src/lib/scoring/engine`)
       - Import `computeInsiderRiskScore` from `@/lib/scoring/insider-risk`
       - Import `DEFAULT_SCORING_CONFIG`, `DEFAULT_INSIDER_RISK_CONFIG` from `@/lib/scoring/config`
       - For each politician:
         a. Get all trades for this politician
         b. Score each trade using `scoreTrade()` with context built from politician data
         c. Score the politician using `scorePolitician()` to get season totals
         d. Compute Insider Trading Risk Score using `computeInsiderRiskScore()` with components derived from trade patterns:
            - donorOverlap: random 0-100 (simulated since we don't have real donor data)
            - suspiciousTiming: % of trades within 7 days of any committee hearing date (simulated)
            - committeeConflict: % of trades in sectors matching their committee jurisdictions (derive from committee names)
            - stockActCompliance: 100 - (% of trades with daysToDisclose > 45) * 100
            - tradeVolume: normalize trade count to 0-100 scale (0 trades = 0, max trades politician = 100)
         e. Assign salary tier and salary cap based on season points:
            - Sort all politicians by seasonPoints descending
            - Top 10%: 'elite' ($8000-$10000)
            - 10-25%: 'starter' ($6000-$7999)
            - 25-50%: 'mid-tier' ($4000-$5999)
            - 50-75%: 'bench' ($2000-$3999)
            - 75-90%: 'sleeper' ($1000-$1999)
            - Bottom 10%: 'unranked' ($500-$999)
       - Exclude trades with incomplete data (missing return values) per D-07. Log excluded trades.
       - Write final `public/data/politicians.json` (full Politician[] conforming to type)
       - Write final `public/data/trades.json` (full Trade[] with scoreBreakdown and fantasyPoints)
       - Handle path aliases: since tsx doesn't natively support @/ aliases from tsconfig, use relative imports like `../src/lib/scoring/engine` OR create a tsconfig.scripts.json that extends the main tsconfig with baseUrl set for scripts, and run as `npx tsx --tsconfig tsconfig.scripts.json scripts/score-all.ts`
